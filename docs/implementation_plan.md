# 구현 계획 - 구글 시트 내보내기 확인 창 디자인 개선

현재 브라우저 기본 `confirm()` 창을 사용하여 투박해 보이는 "데이터 전송 확인" 대화상자를, 크고 세련된 커스텀 모달(Modal)로 교체합니다.

## 변경 제안

### 1. HTML 구조 추가 (`index.html`)
- `body` 태그 내에 새로운 모달 컨테이너 `<div id="confirm-modal" class="portal">`을 추가합니다.
- 모달 내부는 다음과 같이 구성합니다:
    - **헤더**: "구글 시트 저장" (크고 굵은 폰트)
    - **본문**: 전송할 데이터를 요약하여 보여줄 영역. (아이콘 활용)
        - 🏆 상위 우승: [이름]
        - ...
    - **푸터**: [취소] [전송하기] 버튼 (큰 크기, 강조 색상)

### 2. CSS 스타일링
- **모달 디자인**:
    - 화면 중앙 정렬, 둥근 모서리(20px), 부드러운 그림자.
    - 배경 블러 처리 (Glassmorphism 효과 살짝 적용).
- **타이포그래피**:
    - 제목: 24px 이상, Bold.
    - 본문 리스트: 16px 이상, 줄 간격 넉넉하게. 이모지(🏆) 크기 키움.
- **버튼**:
    - 모바일에서도 누르기 편하도록 높이 50px 이상.
    - "전송하기" 버튼은 녹색/파란색 그라디언트로 강조.

### 3. JavaScript 변경
- 기존 `exportToGoogleSheet` 함수 내의 `confirm()` 호출을 제거합니다.
- 대신 `openConfirmModal(data, onConfirm)` 함수를 호출하여 모달을 띄웁니다.
- 모달의 "전송하기" 버튼 클릭 시 실제 `fetch` 로직이 실행되도록 콜백 구조로 변경합니다.

## 예상 화면
- 화면 중앙에 흰색(또는 밝은 회색) 카드가 뜹니다.
- "총 10명의 기록을 저장하시겠습니까?" 문구가 크게 보입니다.
- 우승자 명단이 카드 형태로 깔끔하게 나열됩니다.
- 하단에 꽉 찬 버튼으로 [취소] [확인]이 배치됩니다.

## 기능 추가 제안: 결과 이미지 캡쳐 (Image Export)

사용자가 토너먼트 대진표 결과를 이미지 파일(PNG)로 저장하여 밴드 등에 공유할 수 있도록 합니다.

### 1. 기술 스택
- **Library**: `html2canvas` (CDN 사용)
- 클라이언트 측에서 HTML 요소를 Canvas로 렌더링 후 이미지 파일로 변환.

### 2. 구현 상세
- **HTML (`index.html`)**:
    - `html2canvas` 라이브러리 스크립트 태그 추가.
    - 대시보드 컨트롤 패널에 "📷 이미지 저장" 버튼 추가.
- **JavaScript**:
    - `captureResultImage()` 함수 구현.
    - `#tournament-area` (또는 전체 결과 영역)를 타겟으로 캡쳐.
    - 캡쳐된 Canvas를 `<a>` 태그의 `download` 속성을 이용해 파일로 다운로드 트리거.
- **파일이름**: `토너먼트_결과_YYYY-MM-DD.png` 형식.

### 3. UI/UX
- 이미지 저장 버튼 클릭 시 "이미지 생성 중..." 로딩 표시 (기존 로딩 오버레이 재사용 가능).
- 생성 완료 시 자동으로 다운로드 시작.
